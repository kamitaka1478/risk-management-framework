# RMF（リスクマネジメントフレームワーク学習プラットフォーム）

## プロジェクト概要

このプロジェクトは、**NIST SP 800-37 リスクマネジメントフレームワーク（RMF）** の学習を目的とした、簡易セキュリティ管理プラットフォームです。セキュリティエンジニア向けの実務的な学習システムとして、リスク管理とセキュリティコントロールの実装・追跡を実践できます。

## システムの目的

- **RMF6ステップの理解と実装**：NIST RMFの各フェーズをシステムで学習
- **セキュリティ脆弱性の深い理解**：実装を通じてOWASP Top 10と対策方法を習得
- **本番環境を想定した開発**：認証、認可、監査ログなど実務的なセキュリティ機能を実装

## 主な機能

### Phase 1（基礎）：ユーザー認証・ロール管理
- ユーザー認証（パスワードハッシング、セッション管理）
- ロールベースアクセス制御（RBAC）：管理者、マネージャー、一般ユーザー
- CSRF対策、入力検証

### Phase 2（データ保護）：リスク情報管理
- リスク情報の登録・編集・削除
- リスク評価スコア（CVSS的な簡易計算）
- データベースセキュリティ（SQLインジェクション対策）

### Phase 3（アクセス制御）：セキュリティコントロール追跡
- 実施予定コントロールの管理
- 実施状況の記録と監査
- 最小権限の原則の実装

### Phase 4（監査・ログ）：監視・継続管理
- 全操作の監査ログ記録
- ユーザー操作の追跡可能性
- ログの改ざん防止メカニズム

### Phase 5（本番化）：デプロイとセキュア運用
- HTTPS/TLS対応
- 環境変数によるシークレット管理
- 依存関係の脆弱性チェック

## 技術スタック

| カテゴリ | 技術 | 理由 |
|---------|------|------|
| **言語** | Python 3.9+ | セキュリティライブラリ充実、既存知識を活用 |
| **フレームワーク** | FastAPI | 軽量、型安全、セキュリティベストプラクティスが明示的 |
| **データベース** | PostgreSQL | セキュア、プロダクション的な経験が積める、複雑なクエリ対応 |
| **ORM** | SQLAlchemy | FastAPIとの統合、SQLインジェクション対策 |
| **フロントエンド** | HTML/CSS/JavaScript（または Vue.js） | シンプルで学習負荷を抑える |
| **認証・認可** | python-jose, passlib | JWT、パスワード管理 |
| **デプロイ** | Docker + 環境変数管理 | 本番環境を想定した安全な運用 |

## NIST RMF との対応

このシステムは、NIST SP 800-37 のRMF 6ステップを以下のように実装します：

1. **Categorize（分類）**：システム分類機能
2. **Select（選択）**：セキュリティコントロール選択インターフェース
3. **Implement（実装）**：コントロール実施状況の管理
4. **Assess（評価）**：評価結果の記録機能
5. **Authorize（承認）**：システム承認フロー
6. **Monitor（監視）**：継続的監視・ログ機能

## セキュリティ学習目標

このプロジェクトを通じて以下を習得します：

- ✓ **認証・認可**：パスワード管理、セッション、ロール管理
- ✓ **入力検証とサニタイゼーション**：SQLインジェクション、XSS対策
- ✓ **暗号化とハッシング**：パスワード、通信のセキュアさ
- ✓ **アクセス制御**：RBAC、最小権限の原則
- ✓ **監査・ログ**：改ざん防止、トレーサビリティ
- ✓ **セキュアなデプロイ**：環境管理、依存性チェック
- ✓ **脆弱性管理**：OWASP Top 10の理解と対策
